Linguagens exotéricas-Brainf**k
No mundo da computação as linguagens de programação tem um papel especial, já que por meio delas fomos capazes de "animar" máquinas excepcionais de forma autômata. Reduzindo custos de produção, ao passo que não era mais preciso desenvolver um hardware específico para solucionar cada problema proposto, bastava simplesmente usar um hardware programável e desenvolver o software destinado a usar os recursos de máquina disponíveis, executar as operações pertinentes e finalmente solucionar o problema. Neste pouco tempo tivemos dezenas, centenas de linguagens, cada uma com seus prós e contras. Inteligentemente classificadas em grupos de "afinidade".
Nesse meio surgiram as Esolangs (Esoteric programming language), ou Linguagens exotéricas. Estas linguagens são desenvolvidas não intencionando resolver um problema específico, sequer são desenvolvidas para uso em sistemas comerciais. São linguagens de programação que foram desenvolvidas para testar os limites dos projetos e por que não dizer dos programadores. Podem ser desenvolvidas para aplicar uma teoria específica ou mesmo ser uma brincadeira entre um grupo de estudiosos de linguagens de programação. Estas linguagens não estão preocupadas com simplicidade, velocidade ou mesmo usabilidade. Apesar de serem desprezadas pelos livros técnicos, eu acredito que algumas delas deviam ao menos ser testadas por nós programadores, pois é interessante treinar o raciocínio baseado em linguagens puramente nos oferecem o poder de processamento de uma Máquina de Turing. Dentre as linguagens exotéricas que mais me chamam a atenção encontra a Brainfuck, não vou traduzir o termo pois acredito que todos sejam capazes de entender a brincadeira deste nome.
Na minha opinião foi uma infeliz escolha pois quase que 100% das vezes que se desejar pesquisar o termo na internet, o log do proxy irá associar o seu IP a sites proibidos (kkkkkkk).
Mas, deixando o nome da linguagem um  pouco de lado, esta linguagem foi idealizada para ser extremamente minimalista, dado que possui somente oito caracteres distintos. Foi criada em 1993 por Urban Müller.
Os símbolos aceitos são:
CaractereSignificado
>incrementa o ponteiro de dados (passa a apontar para o próximo a direita).
<decrementa o ponteiro de dados (passa a apontar para a célula a esquerda).
+operador de incremento (igual a ++).
-operador de decremento (igual a --).
.imprime o byte que estiver armazenado no endereço atual.
,aceita um byte como input e armazena no endereço atual do ponteiro de dados.
[inicia um laço, se o byte no endereço atual for zero, busca pelo primeiro comando após  ] e o executa, caso seja maior que zero, inicia um laço e executa todos os comando internos.
]se no endereço atual do ponteiro de dados contiver um valor diferente de zero, ele cria um laço para executar todos os comando contidos dentro do [ anterior.
