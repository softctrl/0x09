Como linguagem escolhida para desenvolvimento eu preferi C++/CLI que está oficialmente especificada no documento ECMA-372 que data de dezembro 2005. Em palavras curtas, C++/CLI é uma especificação de C++ que veio para substituir o set de tecnologias desenvolvido pela Microsoft conhecido como "Managed Extensions for C++" sendo que este último encontra-se obsoleto e foi totalmente substituído pela tecnologia C++/CLI. Bem, eu gosto de lembrar dessa linguagem como uma tentativa bem sucedida de dar à linguagem C++ tudo o que o Dot.Net tem a oferecer e que por sua fez foi bem copiado da plataforma Java :D.

Bem, só o nome dos atributos no enum ninguém merece, nada intuitivos!!! Bem, depois de escolhido o template a ser usado na notificação, vamos ao que interessa, CODIFICAR!!
É importante saber que toda a gama de objetos que iremos utilizar pertencem aos namespace Windows::UI::Notifications e Windows::Data::Xml::Dom. Então se preferir pode importar logo dentro da classe que for usar para fazer uma chamada ao ToastNotification:
... using namespace Windows::UI::Notifications;
using namespace Windows::Data::Xml::Dom;
...
Feito isso, escolha o modelo que mais se adequar ao seu programa, isso se for para Windows 8, ou caso seja Windows Phone escolha qualquer um desde que seja o ToastText02 (kkkkkkkkk).
Bem, como estou me ambientando no desktop pois não tenho um desktop 64bits e nem um smartphone com Windows Phone ainda, eu vou usar o template ToastImageAndText01. Para isso: ... ToastTemplateType toastModelo = ToastTemplateType::ToastImageAndText01;  XmlDocument^ toastXmlDocument = ToastNotificationManager::GetTemplateContent(toastTemplate); ...
O método GetTemplateContent nos devolve um XmlDocument com o XML do modelo escolhido. De posse desse objeto XmlDocument temos agora que alimentar os dados que queremos apresentar para o usuário. Os dados pelo modelo escolhido devem ser uma imagem e um texto. Então vamos lá:
... // Informando a imagem a ser exibida XmlNodeList^ toastImageElements = toastXmlDocument->GetElementsByTagName("image"); // Neste caso eu estou usando uma imagem da internet // mas poderia ser do próprio projeto: static_cast<XmlElement^>(toastImageElements->Item(0))->SetAttribute("src", "http://url_da_imagem"); static_cast<XmlElement^>(toastImageElements->Item(0))->SetAttribute("alt", "graphic"); // Informando o texto a ser exibido XmlNodeList^ toastTextElements = toastXmlDocument->GetElementsByTagName("text"); toastTextElements->Item(0)->InnerText = "Ocorreu um erro no sistema!"; // Informando a duração do Toast IXmlNode^ toastNode = toastXmlDocument->SelectSingleNode("/toast"); static_cast<XmlElement^>(toastNode)->SetAttribute("duration", "long"); // Até que enfim vamos começar a ver uma luz no fim do túnel!! ToastNotification^ toast = ref new ToastNotification(toastXmlDocument); // Demorou mais aqui está: ToastNotificationManager::CreateToastNotifier()->Show(toast); ...
Bem, confesso que não é a complexidade o que eu estou discutindo aqui, pois como vocês puderam ver acima não é complexo, eu estou reclamando é que um processo tão simples quanto mostrar uma mensagem para um usuário deveria ser algo também simples, resolvido em no máximo duas linhas de codificação. Experimente colocar mais coisas, como um som a ser executado ou mesmo usar outro template mais complexo. Na minha opinião, esta tarefa deveria ser resolvida com algo do tipo:
...
ToastNotificationManager::CreateToastNotifier("titulo", "texto", ToastNotificationManager::LONG)->Show();
...
