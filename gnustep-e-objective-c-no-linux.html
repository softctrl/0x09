Objective-C Segundo a Wikipedia, Objective-C é uma linguagem de programação orientada a objetos e de propósito geral que adiciona o estilo de mensagem da Smalltalk para a linguagem C de Dennis Ritchie. A linguagem Objective-C é a principal linguagem utilizada pela Apple nos sistemas operacionais  OS X e iOS bem como nas suas API's de programação. Esta linguagem foi desenvolvida por volta de 1980, e foi selecionada para ser a principal linguagem da NeXT de Steve Jobs, sendo utilizada no sistema operacional NeXTSTEP do qual os sistemas OS X e iOS derivaram. Infelizmente a Apple fecha suas ferramentas de programação de modo que sem um Macbook, se torna praticamente impossível desenvolver para a plataforma da Apple. Mas felizmente, nossa comunidade é bastante ativa, e também não vamos crucificar a Apple, pois a mesma também possui uma (pequena) porção open source. Um fato  importante é que apesar de o GNUstep antigo OpenStep, ser anterior ao Cocoa, o Cocoa foi baseado no OpenStep. Perceba na imagem abaixo que o GNUstep se enquadra no grupo de kits para criação de interface gráfica de usuários.

GNUstep Também segundo a Wikipedia, GNUstep é uma implementação gratuita das bibliotecas Objective-C Cocoa, componentes visuais, e ferramentas de desenvolvimento para Unix-like e Windows, sendo parte integrante do projeto GNU. Foi desenvolvido pelo programador Paul Kunz que atuava no Centro de Aceleração Linear de Stanford. Na época ele recebeu a responsabilidade de portar uma aplicação Hipodraw que era desenvolvido para a plataforma NeXTSTEP, o curioso é que ao invés de rescrever a aplicação para outras plataformas ele decidiu portar toda a camada de objetos do NeXTSTEP do qual o Hipodraw dependia. Foi um sucesso, o projeto pode ser recompilado sem precisar mudar uma linha de seu código. A primeira liberação aberta do projeto, até então chamado de OpenStep se deu em 1994. Neste post irei mostrar como instalar o pacote GNUStep. Este pacote implementa o Core Foundation de bibliotecas do Objective-C sendo que quase 90% dos códigos implementados em um Macbook também poderão ser recompilados no Centos com pouca ou quase nenhuma alteração. Como primeiro passo monte uma distribuição básica Centos, eu sempre uso o cd de instalação mínimo, e vou incluindo os pacotes que preciso ao longo da utilização do pc. Mais neste posto estou assumindo que todos sabemos como instalar e configurar o Linux Centos. Após instalado o Centos daremos início a instalação dos pacotes necessários para compilação do GNUStep. Para organizar a instalação eu costumo criar uma diretório gnustep onde deixo salvo os pacotes que eu fiz download quando fui instalar o GNUstep. ~#mkdir gnustep ~#cd gnustep Agora que estamos no diretório do nosso pequeno projeto, podemos iniciar a configuração do Centos para poder instalar o GNUstep. 1) Atualizar os pacotes do Centos. Isso não é totalmente obrigatório mais como acabamos de instalar o Centos, eu assumo que sempre é bom atualizar os pacotes para as suas versões atuais estáveis: [gnustep]#yum -y upgrade
2) Caso você tenha instalado o Centos Minimal, você não possui as ferramentas de desenvolvimento, make, gcc e outros, mais isso é simples de resolver: [gnustep]#yum -y groupinstall "Development Tools"
3) Agora sim, vamos instalar pacotes obrigatórios para dar prosseguimento a instalação do GNUstep: [gnustep]#yum -y install wget ncurses* openssl* libXt* glib* libjpeg* libpng* libxml* libxslt* libtiff* libXext* gnutls* gcc-objc* libobjc* libffi*
Fora a biblioteca de desenvolvimento ncurses, todo o restante é obrigatório para a instalação do GNUstep. Outro ponto importante é sobre a lib FFI que apesar de estar inclusa no comando acima, a mesa esta com versão desatualizada nos repositórios do Centos, precisamos de no mínimo da versão 3.0.9, então mais a frente iremos baixar os fontes e instalar corretamente esta dependência. Agora vamos instalar via fontes mais dois pacotes necessários para instalação do GNUstep. Neste momento vamos resolver o problema de instalação da biblioteca FFI no Centos: 4) Primeiro baixe os fontes da libffi diretamente do endereço https://sourceware.org/libffi/ [gnustep]#wget -c "ftp://sourceware.org/pub/libffi/libffi-3.0.13.tar.gz"
Descompacte o arquivo utilizando o tar: [gnustep]#tar -zxvf libffi-3.0.13.tar.gz Agora vamos para a instalação. De dentro do diretório descompactado inicie o processo de compilação normal (configure/ make/ make install), tendo atenção de informa o parâmetro --prefix no configure: [libffi-3.0.13]#./configure --prefix=/usr/include [libffi-3.0.13]#make [libffi-3.0.13]#make install Agora neste ponto vamos fazer uma "gambiarra" para que tudo funcione corretamente:
[gnustep]#cp -Rvf /usr/include/lib64/libffi* /usr/lib64/
Isso foi necessário pois o processo make install copiou as lib`s para o diretório /usr/include/lib64/ e nós precisamos deles na /usr/lib64/.
Após este processo sinta-se a vontade para excluir o diretório libffi-3.0.13/.
5) Neste ponto vamos instalar o ICU-International Components for Unicode. O mesmo pode ser instalado via yum, porém prefiro instalar via código fonte:
[gnustep]#wget -c "http://download.icu-project.org/files/icu4c/52rc/icu4c-52_rc2-src.tgz"
Em seguida descompacte o arquivo:
[gnustep]#tar -zxvf icu4c-52_rc2-src.tgz
Agora entre no diretório icu/source e proceda com os comandos para configuração e instalação do projeto:
[source]#./configure
[source]#gmake
[source]#gmake install
Este processo poderá demorar um pouco, então eu sugiro irmos tomar um café! :)
Terminado o processo de compilação e instalação do pacote ICU, faremos mais um ajuste igual ao que fizemos para a libffi.
[source]#cp -Rvf /usr/local/lib/libicu* /usr/lib64/
Após este processo sinta-se a vontade para excluir o diretório icu/.
Pronto agora podemos baixar os pacotes do GNUStep, compilar, instalar e somente então começar a desenvolver em Objective-C no Linux.
6) Vamos instalar o primeiro pacote do GNUstep, o pacote Startup
[gnustep]#wget -c "ftp://ftp.gnustep.org/pub/gnustep/core/gnustep-startup-0.32.0.tar.gz"
Descompacte o arquivo:
[gnustep]#tar -zxvf gnustep-startup-0.32.0.tar.gz
De dentro do diretório descompactado, execute o script InstallGNUstep:
[gnustep-startup-0.32.0]#./InstallGNUstep
Pressione enter para continuar. O script irá validar todas as dependências e lhe pedirá mais uma vez para pressionar enter para continuar.

Agora a base do GNUstep está instalada no nosso ambiente. Ainda faltam alguns pacotes a serem instalados mais se chegamos até aqui, dificilmente haverão problemas no nosso caminho. A partir deste momento irei descrever somente passos que fogem ao padrão de uma instalação Linux por meio do código fonte. 7) Vamos instalar o pacote GNUstep Make: Baixe o arquivo gnustep-make-2.6.6.tar.gz, descompacte e inicie o processo de configuração/compilação e instalação. [gnustep]#wget -c "ftp://ftp.gnustep.org/pub/gnustep/core/gnustep-make-2.6.6.tar.gz" [gnustep]#tar -zxvf gnustep-make-2.6.6.tar.gz [gnustep]#cd gnustep-make-2.6.6 [gnustep-make-2.6.6]#./configure [gnustep-make-2.6.6]#make [gnustep-make-2.6.6]#make install 8) Agora o pacote GNUstep Base: Baixe o arquivo gnustep-base-1.24.6.tar.gz, descompacte e inicie o processo de configuração/compilação e instalação.
[gnustep]#wget -c "ftp://ftp.gnustep.org/pub/gnustep/core/gnustep-base-1.24.6.tar.gz"
[gnustep]#tar -zxvf gnustep-base-1.24.6.tar.gz
[gnustep]#cd gnustep-base-1.24.6
Atenção, informar o --prefix=/usr/include é obrigatório. Aqui também teremos tempo de ir tomar um café, pois a compilação também é um pouco demorada.
[gnustep-base-1.24.6]#./configure --prefix=/usr/include
[gnustep-base-1.24.6]#make
Como um passo adicional, mais não obrigatório, podemos realizar os testes com o comando seguinte:
[gnustep-base-1.24.6]#make check
[gnustep-base-1.24.6]#make install
Aqui também temos de tomar o cuidado de copiar as bibliotecas para a pasta /usr/lib64, pois caso contrário irá nos atrapalhar nos processos seguintes.
[gnustep-base-1.24.6]#cp -Rvf /usr/local/lib/libgnustep-base* /usr/lib64/
9) Vamos instalar o pacote GNUstep GUI: Baixe o arquivo gnustep-gui-0.24.0.tar.gz, descompacte e inicie o processo de configuração/compilação e instalação.
[gnustep]#wget -c "ftp://ftp.gnustep.org/pub/gnustep/core/gnustep-gui-0.24.0.tar.gz"
[gnustep]#tar -zxvf gnustep-gui-0.24.0.tar.gz
[gnustep]#cd gnustep-gui-0.24.0
[gnustep-gui-0.24.0]#./configure
[gnustep-gui-0.24.0]#make

Como um passo adicional, mais não obrigatório, podemos realizar os testes com o comando seguinte:
[gnustep-gui-0.24.0]#make check
[gnustep-gui-0.24.0]#make install
10) Vamos instalar o pacote GNUstep Back: Baixe o arquivo gnustep-back-0.24.0.tar.gz, descompacte e inicie o processo de configuração/compilação e instalação.
[gnustep]#wget -c "ftp://ftp.gnustep.org/pub/gnustep/core/gnustep-back-0.24.0.tar.gz"
[gnustep]#tar -zxvf gnustep-back-0.24.0.tar.gz
[gnustep]#cd gnustep-back-0.24.0
[gnustep-back-0.24.0]#./configure
[gnustep-back-0.24.0]#make
[gnustep-back-0.24.0]#make install
11) Vamos instalar o pacote GNUstep Corebase:

Baixe o arquivo gnustep-back-0.24.0.tar.gz, descompacte e inicie o processo de configuração/compilação e instalação.
[gnustep]#wget -c "ftp://ftp.gnustep.org/pub/gnustep/libs/gnustep-corebase-0.1.tar.gz"
[gnustep]#tar -zxvf gnustep-corebase-0.1.tar.gz
[gnustep]#cd gnustep-corebase-0.1
[gnustep]#./configure

Como um passo adicional, mais não obrigatório, podemos realizar os testes com o comando seguinte:
[gnustep-corebase-0.1]#make check
[gnustep-corebase-0.1]#make
[gnustep-corebase-0.1]#make install
Agora sim, concluímos toda a instalação do pacote GNUstep, basta simplesmente adicionar uma variável de ambiente para que o processo de make seja menos traumático :D
[gnustep]#echo "export GNUSTEP_MAKEFILES=/usr/GNUstep/System/Library/Makefiles" >> /etc/profile
